# 163邮箱自动搜索截图工具使用说明

本文档用于把工具放到另一台 Windows 电脑上进行真实测试。

## 1. 工具用途

本工具用于批量处理 Excel 中的邮件名称：

1. 读取 Excel 第一列的邮件名称。
2. 打开 163 邮箱。
3. 等待用户手动扫码或账号登录。
4. 按 Excel 中的原始内容逐条搜索邮件。
5. 打开搜索结果中的邮件详情。
6. 从邮件标题区域截图到下方二维码区域。
7. 按 Excel 原始内容保存 PNG 图片。
8. 生成任务报告。

示例 Excel 内容：

```text
结题通知_LC-P20251125017-3_外泌体鉴定(粒径)_LC-C202511250016-贾烨玮_小鼠_260601
```

对应截图文件名：

```text
结题通知_LC-P20251125017-3_外泌体鉴定(粒径)_LC-C202511250016-贾烨玮_小鼠_260601.PNG
```

说明：如果名称中包含 Windows 不允许的文件名字符，例如 `/ \ : * ? " < > |`，工具会自动替换成 `_`。

## 2. 目标电脑运行环境

如果只是运行已经打包好的 exe，目标电脑不需要安装 Python。

目标电脑需要：

- Windows 10 或 Windows 11
- Google Chrome 浏览器
- 能正常访问 163 邮箱
- 可登录的 163 邮箱账号
- Excel 文件，格式建议为 `.xlsx`
- 截图保存目录有写入权限

不需要安装：

- Python
- PySide6
- Playwright
- openpyxl
- PyInstaller

这些依赖已经打包进 exe。

## 3. 拷贝哪些文件到另一台电脑

最简单方式：只拷贝这个文件即可：

```text
dist/MailAutoScreenshot.exe
```

建议在目标电脑新建一个目录，例如：

```text
D:/MailAutoScreenshot/
```

然后把 exe 放进去：

```text
D:/MailAutoScreenshot/MailAutoScreenshot.exe
```

首次运行后，exe 同级目录会自动生成：

```text
D:/MailAutoScreenshot/config/config.json
D:/MailAutoScreenshot/logs/app.log
```

不要把 exe 放到需要管理员权限写入的目录，例如：

```text
C:/Program Files/
```

否则配置和日志可能无法保存。

## 4. Excel 文件要求

Excel 第一列放邮件名称。

示例：

| A列 |
|---|
| 结题通知_LC-P20251125017-3_外泌体鉴定(粒径)_LC-C202511250016-贾烨玮_小鼠_260601 |
| 订单001 |
| 测试邮件 |

要求：

- 从第一列读取。
- 空行会自动跳过。
- 不需要表头；如果第一行是表头，也会被当成搜索内容。
- 文件格式建议使用 `.xlsx`。
- 支持 `.xlsx`、`.xlsm`、`.xltx`、`.xltm`。

建议首次测试时只放 3 到 5 条数据，确认效果后再跑 400 到 500 条。

## 5. 首次运行步骤

1. 双击运行：

```text
MailAutoScreenshot.exe
```

2. 在界面中选择：

- Excel 文件
- 图片保存目录
- Chrome 用户目录

默认 Chrome 用户目录：

```text
C:/MailAutoProfile
```

这个目录用于保存 163 邮箱登录状态。第一次登录成功后，下次通常不需要重新登录。

3. 点击：

```text
开始任务
```

4. 工具会启动 Chrome 并打开 163 邮箱。

5. 如果没有登录，工具会暂停等待。请在打开的 Chrome 页面中手动扫码或账号登录。

6. 登录完成后，工具会继续处理 Excel 中的邮件名称。

## 6. 运行过程中如何判断是否正常

界面会显示：

- 总数量
- 当前处理
- 成功数量
- 失败数量
- 进度条
- 实时日志

每处理一条，日志会显示成功或失败原因。

单条失败不会导致整个任务退出，工具会记录失败并继续下一条。

## 7. 暂停、继续、停止

按钮说明：

- `开始任务`：开始读取 Excel 并执行自动化。
- `暂停`：当前步骤结束后暂停下一条处理。
- `继续`：从暂停处继续。
- `停止`：当前步骤结束后停止剩余任务。

注意：如果浏览器正在等待网页加载或截图，停止可能不会立即生效，会在当前操作结束后退出。

## 8. 输出结果

截图保存到你在界面选择的保存目录。

截图命名规则：

```text
Excel原始内容.PNG
```

示例：

```text
结题通知_LC-P20251125017-3_外泌体鉴定(粒径)_LC-C202511250016-贾烨玮_小鼠_260601.PNG
```

如果 Excel 中出现重复名称，为避免覆盖，后续文件会自动追加编号：

```text
邮件名称.PNG
邮件名称_2.PNG
邮件名称_3.PNG
```

任务完成后，会在截图保存目录生成报告：

```text
task_report_YYYYMMDD_HHMMSS.csv
```

报告包含：

- keyword：Excel 中的邮件名称
- status：success 或 failed
- file_path：成功截图路径
- error：失败原因

## 9. 日志文件

日志文件在 exe 同级目录：

```text
logs/app.log
```

如果程序运行异常、搜索失败、截图失败，可以先看这个文件。

## 10. 配置文件

配置文件在 exe 同级目录：

```text
config/config.json
```

默认内容类似：

```json
{
  "chrome_profile": "C:/MailAutoProfile",
  "timeout": 20,
  "save_path": "D:/MailScreenshot",
  "mail_url": "https://mail.163.com/"
}
```

字段说明：

- `chrome_profile`：Chrome 登录状态保存目录。
- `timeout`：网页元素等待超时时间，单位秒。
- `save_path`：默认截图保存目录。
- `mail_url`：163 邮箱地址。

通常不需要手动修改，界面中选择目录后会自动保存。

## 11. 真实测试建议

建议按这个顺序测试：

1. 准备一个只有 1 条邮件名称的 Excel。
2. 启动 exe。
3. 选择 Excel 和截图目录。
4. 点击开始任务。
5. 完成 163 登录。
6. 检查是否能搜索到邮件。
7. 检查是否进入邮件详情。
8. 检查截图是否包含标题到二维码区域。
9. 检查图片文件名是否等于 Excel 原始内容。
10. 再测试 5 到 10 条。
11. 最后测试 400 到 500 条。

## 12. 常见问题

### 12.1 双击 exe 没反应

检查：

```text
logs/app.log
```

如果 exe 放在没有写入权限的目录，请移动到例如：

```text
D:/MailAutoScreenshot/
```

### 12.2 提示找不到 Chrome

目标电脑需要安装 Google Chrome。

安装后重新运行 exe。

### 12.3 每次都需要重新登录

检查 Chrome 用户目录是否固定，例如：

```text
C:/MailAutoProfile
```

不要每次换不同目录。

不要删除这个目录。

### 12.4 搜索不到邮件

可能原因：

- Excel 中邮件名称和 163 邮箱实际标题不完全匹配。
- 163 邮箱搜索结果加载较慢。
- 网络不稳定。
- 163 邮箱页面结构变化。

可以尝试：

- 手工在 163 邮箱搜索同一条内容。
- 把 `config/config.json` 中的 `timeout` 从 `20` 改成 `40`。
- 查看 `logs/app.log` 中的失败原因。

### 12.5 截图没有截到二维码

工具优先从标题截到二维码底部。

如果 163 页面里的二维码不是普通图片或二维码元素无法识别，工具会降级为：

1. 截邮件详情区域。
2. 再失败则截整页。

如果测试发现二维码经常没有被截到，需要根据真实页面补充二维码选择器。

### 12.6 文件名和 Excel 不完全一致

Windows 不允许文件名中出现这些字符：

```text
< > : " / \ | ? *
```

工具会把这些字符替换为 `_`。

除此之外会尽量保留 Excel 原始内容。

### 12.7 运行过程中可以操作电脑吗

可以，但不建议频繁操作自动化 Chrome 窗口。

不要关闭工具打开的 Chrome。

不要在任务运行时手动切换 163 邮箱页面。

## 13. 安全注意事项

`C:/MailAutoProfile` 中会保存 Chrome 登录状态。

不要把这个目录复制给别人。

不要把这个目录上传到 GitHub。

如果需要清除登录状态，可以删除：

```text
C:/MailAutoProfile
```

删除后下次运行需要重新登录 163 邮箱。

## 14. 如果需要源码运行

源码运行用于开发调试，不是普通使用方式。

需要安装：

- Python 3.11+
- Google Chrome

安装依赖：

```powershell
py -3.11 -m pip install -r requirements.txt
```

运行：

```powershell
py -3.11 MailAutoScreenshot/main.py
```

## 15. 如果需要重新打包 exe

在项目根目录执行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/build_exe.ps1
```

如果依赖已经安装过：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/build_exe.ps1 -SkipInstall
```

打包结果：

```text
dist/MailAutoScreenshot.exe
```

## 16. 当前版本已知边界

- 163 邮箱页面结构如果变化，可能需要调整 `services/selectors.py`。
- 搜索结果默认打开第一个匹配项。
- 截图区域依赖邮件详情页中的标题和二维码元素。
- 如果邮件正文在 iframe 或特殊容器中，截图区域可能需要进一步适配。
- 大批量任务建议先小样本验证选择器和截图效果。
